# pysolarsim

Physics-based simulation of wearable solar harvester signals for motion sensing.

SolarSim synthesises photovoltaic power traces directly from motion-capture
trajectories under a configurable illumination model. It is training-free:
there is no network to fit and no paired recordings needed beyond a one-off
calibration per environment.

It takes exactly two inputs:

1. **A motion trajectory** — timestamped 3-D cell positions and outward
   normals, read through a configurable adapter.
2. **An illumination environment** — the geometry, position, and orientation
   of every light source, authored as a scene file.

Because the radiometric kernel depends only on geometry, everything
hardware-specific (cell efficiency and area, source intensity, spectral
response) collapses into one calibration scalar `kappa` per emitter. Moving to
a new room means editing a scene file. Moving to a new harvester means
refitting `kappa`. Neither requires retraining anything.

## Quick start

```bash
# What does this lighting setup look like?
pysolarsim describe --scene configs/scene/baseline_circular_6000K_40klx.yaml

# Fit kappa once for the environment, from ~60 s of paired motion and PV.
pysolarsim calibrate \
  --scene configs/scene/baseline_circular_6000K_40klx.yaml \
  --mocap configs/mocap/leap_csv.yaml \
  --input "data/baseline_15participants/real" \
  --reference "data/baseline_15participants/real/P01/Free Move__Synchronized.csv" \
  --max-seconds 60 --out results/kappa.json

# Synthesise power for a whole dataset, mirroring the input tree.
pysolarsim simulate --config configs/run/baseline.yaml

# Score simulated traces against the measured PV in the same recordings.
pysolarsim validate \
  --scene configs/scene/geometry_rectangular_6000K_40klx.yaml \
  --mocap configs/mocap/leap_csv.yaml \
  --input data/geometry_rectangular_6000K_40klx \
  --calibration-mode per_recording \
  --report results/fidelity/rect.csv
```

Single recording or whole directory — `--input` accepts either.

## Python API

```python
from pysolarsim import Scene, RecordingSpec, calibrate_scene, simulate_recording

scene = Scene.load("configs/scene/baseline_circular_6000K_40klx.yaml")
spec = RecordingSpec.load("configs/mocap/leap_csv.yaml")

fit = calibrate_scene(scene, ["data/baseline_15participants/real/P01/Free Move__Synchronized.csv"], spec, max_seconds=60)
result = simulate_recording("data/baseline_15participants/real/P02/Circle__Synchronized.csv", scene, spec, kappas=fit.kappas)

print(result.power_uW)   # simulated trace, microwatts
print(result.metrics)    # pearson, rmse, nrmse, ... when the file has real PV
```

## Configuring the illumination environment

A scene file lists emitters. Add a second entry and multi-emitter
superposition happens automatically, with NNLS fitting one `kappa` per source.
Mixed geometries in one scene are supported.

```yaml
units: mm
emitters:
  - id: overhead_disc
    shape: disc              # disc | rectangular | spherical
    position: [-159.81, 868.82, 168.23]
    normal: [0, -1, 0]       # outward; defaults to straight down
    dimensions: {diameter: 200.0}
    kappa: null              # null means "fit me"
    enabled: true
integration:
  disc: {n_r: 20, n_phi: 20}
  rectangular: {n_u: 20, n_v: 20}
calibration:
  method: auto               # auto | scalar_ls | nnls
  mode: reference            # reference | per_recording | fixed
  reference: {max_seconds: 60}
```

`configs/scene/` holds a scene for every illumination configuration in the
paper, generated by `tools/build_scenes.py`.

## Configuring the motion input

The mocap config maps whatever column names, units, and sign conventions your
capture system uses onto the canonical trajectory. Supporting Vicon,
OptiTrack, or a Xsens export is a config edit, not a code change.

```yaml
columns:
  time: time
  position: [leap_palm_x, leap_palm_y, leap_palm_z]
  normal: [leap_palm_normal_x, leap_palm_normal_y, leap_palm_normal_z]
units: {position: mm, time: s}
normal_convention: negate    # Leap reports the palm normal; the cell faces the other way
sample_rate_hz: 100
dropout: {drop_zero_position: true, drop_na: true}
power:                       # only needed for calibration and fidelity scoring
  mode: v_times_i            # v_times_i | column | none
  columns: {voltage: v_dc, current: i_dc}
  units: w
```

## Calibration modes

| Mode | Where `kappa` comes from | Use when |
|---|---|---|
| `reference` | Fitted once from a short paired recording, reused everywhere | Generating a dataset (the paper's protocol) |
| `per_recording` | Refitted on every file | Fidelity checks — `rho` is scale-invariant, so this isolates trace shape |
| `fixed` | Read from the scene file | No paired PV available; shape matters, absolute level does not |

## Output

`simulate` writes one file per input, preserving every column except the
measured PV and adding `P_sim_uW`. Downstream code that consumed real
recordings consumes simulated ones unchanged.

## Layout

| Module | Responsibility |
|---|---|
| `kernel.py` | The shared radiometric kernel and per-geometry quadrature |
| `scene.py` | The illumination environment (input #2) |
| `motion.py` | `MotionTrack`, the contract between ingestion and physics |
| `io/recording.py` | Configurable adapters for motion and power (input #1) |
| `engine.py` | View-factor design matrix and superposition |
| `calibrate.py` | Fitting `kappa` by least squares or NNLS |
| `metrics.py` | Pearson, RMSE, nRMSE, SSIM |
| `pipeline.py` | End-to-end orchestration, batching, writing |
| `cli.py` | Command-line interface |
| `convert.py` | Migrating recorded layout notes into scene files |
